## 🚀 核心亮点：回测性能 30×–100× 提升

v0.4.5 对回测引擎进行了**全链路热点消除**，通过**三层性能档位**让策略回测速度实现数量级飞跃。在典型场景（5 指标 × 4 数据源 × 10 万根 K 线）下，**总耗时从数十分钟降至数十秒**，提升幅度 **30×–100×**。

### 三层性能档位

| 档位 | 适用场景 | 典型提速 | 使用方式 |
|------|---------|---------|---------|
| **Pandas 普通档** | 快速验证、原型开发 | 基准 1× | `api.get_close().rolling(20).mean()` |
| **ndarray 中档** | 已定型策略、减少拷贝 | **5×–10×** | `api.get_close_array(window=20)` |
| **IndicatorCache v2 高性能档** | 生产级回测、参数优化 | **30×–100×** | `api.register_indicator()` + `api.get_indicator()` |

### 底层优化详情（7 大工程点）

1. **P1 – ndarray 价格缓存（DataSource 层）**  
   `set_data()` / `align_data()` 后将 `close/open/high/low/volume` 预转为 `numpy.ndarray`。主循环彻底弃用昂贵的 `data.iloc[i]['close']` Pandas 标签索引（占原主循环 ~12% CPU），改为 **O(1) 数组下标访问**。

2. **P2 – 增量账户状态（Account 层）**  
   `add_trade()` 时以 **O(1)** 增量维护 `_acct_long_pos`、`_acct_long_avg_price`、`_acct_close_profit` 等字段。彻底淘汰原 `get_runtime_account_snapshot()` 中每次 O(T) 重放全部成交列表的慢路径。

3. **P3 – 主循环热点消除（BacktestCore 层）**  
   `backtest_core.py` 主循环内对拥有 `_has_array_cache` 的数据源直接读取 `_price_arr[i]`、`_index_obj[i]`，不再进入 iloc 兜底分支。

4. **P4 – per-Bar K 线缓存（API 层）**  
   同一根 K 线内、同一 `window` 的 `get_klines()` 调用共享一份 DataFrame 视图缓存；`get_close/get_open/get_high/get_low/get_volume` 直接复用该视图取列，避免每次重新 `iloc[start:end]` 切片。

5. **P5 – 权益曲线纯对象计算（Results 层）**  
   `backtest_results.py` 引入 `_EquityState`（`__slots__` 纯 Python 对象），替代原基于 dict/DataFrame 的权益曲线循环。状态读写从 C 层属性查找完成，**消除字符串 hash 与 Boxing 开销**。

6. **P6 – 日志句柄及时关闭**  
   参数优化（`optimize_parameters`）时 `prepare_log_file` 的句柄在每轮结束后主动关闭，避免千轮网格搜索导致句柄泄漏与 IO 阻塞。

7. **P7 – IndicatorCache v2（指标预计算）**  
   在 `initialize(api)` 中一次性注册指标函数，框架在数据加载后**全量预计算**；`strategy(api)` 内通过 `get_indicator()` / `get_indicator_array()` **O(1) 标量/数组查询**。  
   - 回测：`set_data()` 后立即一次性计算完整历史数组  
   - SIMNOW/实盘：每根新 K 线写入后基于 deque 内容全量重算（`maxlen ≤ 1000`，~ms 级）  
   - **数值与普通 Pandas 路径逐位等价**，已通过 `profiling/audit_examples_equivalence.py` 逐笔对账验证。

### 示例对比

```python
# ========== 普通版（Pandas，每根 K 线 O(N)）==========
def strategy(api):
    close = api.get_close()
    if len(close) < 20:
        return
    ma20 = close.rolling(20).mean().iloc[-1]   # 每根 K 线重复计算

# ========== 高性能版（IndicatorCache v2，O(1)）==========
def initialize(api):
    api.register_indicator('ma20',
        lambda c, o, h, l, v: pd.Series(c).rolling(20).mean().to_numpy(),
        window=20)

def strategy(api):
    ma20 = api.get_indicator('ma20')            # 直接数组查表，O(1)
```

## 📦 本地数据模式（无需会员账号）

回测新增 `data_source_mode='local'`，可直接从本地 SQLite (`data_cache/backtest_data.db`) 读取 K 线/Tick 数据，**无需 quant789 俱乐部会员账号**。

- **TICK 回测强制使用 `'local'`**（远程服务器不支持 TICK 推送）
- 支持导入格式：**CSV / Excel / JSON / Parquet / Feather / Pickle**
- 远程模式 (`'data_server'`) 仍保留，鉴权失败时给出清晰的二选一提示（申请会员 或 切本地模式）
- `UnifiedStrategyRunner` 在本地模式下**跳过 `verify_auth()`**，避免缓存污染与不必要的网络请求

## 🛠 策略 API 新增接口

### IndicatorCache v2（方式一：最高性能）

```python
api.register_indicator(name, func, window=None, index=0)   # 注册并预计算
api.unregister_indicator(name, index=0)                    # 注销
api.get_indicator(name, index=0) -> float                  # 当前 Bar 标量值
api.get_indicator_array(name, window=None, index=0) -> np.ndarray  # 最近 window 个值
```

### NumPy 零拷贝直读（方式二：中性能）

```python
api.get_close_array(window=None, index=0) -> np.ndarray
api.get_open_array(window=None, index=0) -> np.ndarray
api.get_high_array(window=None, index=0) -> np.ndarray
api.get_low_array(window=None, index=0) -> np.ndarray
api.get_volume_array(window=None, index=0) -> np.ndarray
```

### 初始化钩子

```python
def initialize(api: StrategyAPI):
    """数据加载后、主循环前调用一次"""
    api.log("策略初始化完成")

runner.run(strategy=strategy, initialize=initialize)
```

框架自动将 `initialize` 挂到 `strategy.initialize`，策略内部可直接 `api.register_indicator(...)`。

## 🔧 实盘 / SIMNOW 改进

1. **IndicatorCache v2 支持实盘/SIMNOW**  
   `LiveDataSource` 实现与回测完全相同的 `register_indicator` / `get_indicator` / `get_indicator_array` 接口。同一套策略代码回测通过后可**零改动**直接上盘。

2. **智能追单 Bug 修复**  
   订单超时撤单后重试计数不再被覆盖（原逻辑导致第 2 次及以后重发始终被视为第 1 次）。

3. **撤单回报兜底键**  
   CTP 撤单回报偶发 `OrderSysID` 为空，提前注册 `FrontID:SessionID:OrderRef` 备用键，确保超时重发链路可靠。

4. **历史预加载增强**  
   `historical_preloader.py` 支持 `kline_source` 参数透传，local/data_server 双路径预加载行为一致。

## ⚠️ 升级注意事项

1. **Python 版本**：继续支持 3.9–3.14；若使用 IndicatorCache v2，推荐 3.10+ 以获得更好的 NumPy 性能。
2. **策略兼容性**：普通 Pandas 写法**完全向后兼容**，无需修改任何旧策略。
3. **高性能迁移**：建议将已有策略复制一份为 `*_高性能.py`，仅把 `rolling/ewm/iloc` 替换为 `register_indicator` + `get_indicator_array`。
4. **本地数据库**：首次使用 `data_source_mode='local'` 时，先运行 `examples/A_工具_导入数据库DB示例.py` 建立 `data_cache/backtest_data.db`。
5. **TICK 回测**：v0.4.5 起 TICK 回测必须指定 `data_source_mode='local'`。
