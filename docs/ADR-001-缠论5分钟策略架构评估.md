# ADR-001: 缠论 5 分钟周期策略 — 基于 SSQuant 现有框架的实现路径

**Status:** Proposed
**Date:** 2026-05-07
**Decider:** 量化交易员（项目 Owner）
**Deadline:** 2026-05-08

---

## Context（背景）

用户希望在现有 SSQuant 项目（`D:\repo\ssquant`）上开发缠论量化策略，要求：

- **周期：** 5 分钟 K 线
- **数据源：** SQLite 数据库 `D:\repo\ssquant\data_cache\kline_data.db`
- **理论参考：** waditu/czsc 库（缠论分析引擎）
- **目标：** 精准识别缠论买卖点，回测 + K 线图可视化

### 已查明的项目现状（关键事实）

| 模块 | 状态 |
|---|---|
| `examples/B_缠论多空信号策略.py` | **已存在** — 1500 行完整实现，集成 czsc，四类信号检测，ATR/结构/中枢三重止损，已带蜡烛图 + 笔 + 中枢 + 信号点完整可视化 |
| 默认 K 线周期 | **15m**（需调到 5m） |
| 默认数据源 | API（`api_data_fetcher`）→ CSV 回退（`au888_1h_real.csv`） |
| `ssquant/data/local_data_loader.py` | **只支持 CSV / HDF5 / 7z，不支持 .db** ← 这是缺口 |
| `ssquant/data/historical_preloader.py` | 已实现 SQLite 读取，但**仅用于实盘/SIMNOW 模式预加载**，不接回测主路径 |
| 数据库表名约定 | `{symbol}_{period}_{adjust}`，例如 `au888_5m_hfq` |
| 测试 | 已有 `test_chanlun_strategy.py` 和未追踪的 `test_chanlun_e2e.py` |

### 决策驱动力

1. **截止日期紧（明天）** — 24 小时内要能跑通回测 + 出图
2. **缠论实现复杂** — 笔/线段/中枢/递归走势识别若从零写易错且无法在期限内验证正确性
3. **用户已用过 czsc** — 现有代码证明依赖路径可行
4. **5m 周期信号噪声更高** — 比 15m 更容易出假信号，参数和冷却期需要重新调

---

## Decision（决策）

**复用现有 `B_缠论多空信号策略.py`，仅做四处针对性改动**：将 K 线周期切到 5m、把 SQLite 数据源接入回测路径、调优 5m 周期参数、保留并增强可视化。**不重写策略本体，不重新实现 czsc。**

---

## Options Considered（候选方案）

### 方案 A：复用现有策略 + 增量适配（推荐）

| 维度 | 评估 |
|---|---|
| 复杂度 | 低 — 4 处改动 |
| 成本 | 半天工作量 |
| 可扩展性 | 中 — 信号逻辑沿用现有四类形态 |
| 团队熟悉度 | 高 — 用户已写过这套代码 |
| 风险 | SQLite 接入需要扩展 `local_data_loader` |

**Pros**
- 24 小时内可跑通
- 信号逻辑（四类形态 + 优先级聚合 + 三重止损）已在 15m 周期验证过
- 可视化模块（蜡烛、笔、线段、中枢矩形、买卖点标注）已现成
- czsc 库提供权威笔/线段计算，避免自研出错

**Cons**
- 5m 周期假信号更多 — 必须重调 `min_bi_len`、`signal_cooldown`、ATR 倍数
- SQLite 加载需要新加分支或预导出 CSV

### 方案 B：从零实现自研缠论引擎

**Pros**
- 完全可控，无外部依赖
- 可针对 5m 周期定制笔/线段判定规则

**Cons**
- 缠论包含包含关系处理、分型确认、笔的最小 K 线数、线段特征序列、中枢扩展等数十条规则 — 在 24 小时内实现并校验正确性**几乎不可能**
- 无法在期限内交付

### 方案 C：换框架（vnpy / backtrader + czsc）

**Pros**
- 部分框架对 SQLite 原生友好

**Cons**
- 抛弃 SSQuant 的三模式统一接口（BACKTEST / SIMNOW / REAL_TRADING）
- 用户已深度投入 SSQuant 配置、CTP 集成、可视化
- 工作量远超期限

---

## Trade-off Analysis（权衡分析）

核心权衡是**正确性 vs 工期**。缠论是一套形式化但边界条件多的理论 — 自研引擎在 24 小时内交付的"正确性"成本远高于复用 czsc 的"灵活性"成本。方案 A 把风险压在三件事上：(a) 5m 参数调优，(b) SQLite 接入，(c) 假信号过滤 — 这三件都是**局部、可测、可回滚**的改动。

5m 周期相比 15m 的核心差异是噪声与信号频次：相同 ATR 倍数下止损会更频繁触发；`min_bi_len=7` 在 5m 上会让笔过于碎；`signal_cooldown=3` 在 5m 上仅相当于 15 分钟冷却，应放大。

---

## Consequences（后果）

**变得更容易**
- 5m 信号策略可在 1 天内出回测结果和图表
- 后续若要改用其他周期（1m / 30m），只需调一个参数

**变得更难 / 待解决**
- `local_data_loader.py` 增加 SQLite 分支后需要回归测试，确保不影响 CSV 路径
- 5m 周期假信号过滤可能需要追加成交量/波动率过滤，下一轮迭代再做
- czsc 库版本锁定（0.9.51）— 如果将来升级需要回归全部信号检测逻辑

**需要重新评估的时点**
- 30 天后：5m 周期回测胜率与盈亏比是否达预期
- 若胜率 < 40% 或夏普 < 1.0 — 重新评估是否需要叠加多周期共振或机器学习过滤

## Action Items（行动清单）

### P0 — 24 小时内完成

1. [ ] **新建 `examples/B_缠论5分钟策略.py`**（基于现有 `B_缠论多空信号策略.py` 复制改造），不直接改原文件以保留 15m 版本
2. [ ] **数据源接入：** 二选一
   - **快速路径：** 写一个一次性脚本 `examples/A_工具_导出DB到CSV.py`，从 `kline_data.db` 读出 5m 数据 → CSV → 走现有 CSV 路径
   - **正经路径：** 在 `ssquant/data/local_data_loader.py` 增加 `.db` 扩展名分支，参数解析 `?table=xxx` 或新增 `db_table` 参数
3. [ ] **参数调整（5m 周期专用）：**
   ```python
   strategy_params = {
       'min_bi_len': 5,              # 15m 用 7，5m 缩短到 5
       'signal_cooldown': 6,         # 15m 用 3，5m 放大到 6（≈30 分钟）
       'atr_period': 14,             # 保持
       'atr_stop_multiplier': 2.5,   # 15m 用 2.0，5m 放宽到 2.5 避免过度止损
       'kline_period': '5m',         # 关键改动
       'base_volume': 1,
       'use_structure_stop': True,
       'v_reversal_power_ratio': 1.3, # 15m 用 1.2，5m 提高门槛
       'golden_ratio': 1.618,
   }
   ```
4. [ ] **回测配置：** `kline_period='5m'`，`adjust_type='1'`（后复权），合约 `au888` 或用户实际品种
5. [ ] **跑一次回测，检查输出：** 信号 CSV、JSON、缠论分析图（`backtest_results/chanlun_*.png`）

### P1 — 第二周

6. [ ] **5m 假信号过滤：** 在 `aggregate_signals` 后加成交量阈值过滤（当前笔成交量 < N 期均量 × 0.8 则忽略）
7. [ ] **回归测试：** `pytest tests/test_chanlun_strategy.py -v`，确保现有用例不破
8. [ ] **新增 5m 端到端测试：** 复用 `test_chanlun_e2e.py` 模式，用最近 1 个月真实数据跑一遍信号点对比

### P2 — 第二个月

9. [ ] **多周期共振：** 利用 SSQuant 的多数据源机制，在 5m 出信号时用 15m / 30m 同向确认才下单
10. [ ] **参数优化：** 用 `parameter_optimizer.py` 网格搜索 `(min_bi_len, atr_stop_multiplier, signal_cooldown)`

## 风险与回滚

| 风险 | 检测信号 | 回滚动作 |
|---|---|---|
| SQLite 加载改动破坏 CSV 路径 | 现有 `B_*.py` 例子运行报错 | 将 `local_data_loader.py` 改动隔离到独立函数，主路径不动 |
| 5m 信号频次过高导致频繁交易 | 单日信号 > 20 条 | 把 `signal_cooldown` 调到 12，叠加成交量过滤 |
| czsc 增量更新性能瓶颈 | 单根 K 线处理 > 100ms | 改为每 N 根 K 线全量重建一次 |
