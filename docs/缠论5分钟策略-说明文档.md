## 一、策略一句话总结

在 **5 分钟 K 线** 上用 [czsc 库](https://github.com/waditu/czsc) 识别缠论的四类买卖点信号（1 类转向 / 2 类强势延续 / 3 类强势确认 / 4 类 V 反），再叠加 **长周期趋势过滤**（5m 内部合成 MA240）和 **缠论"同走势 ≤ 2 次"原则**，最后用 ATR / 结构 / 中枢三重止损保护。

属于典型 **低胜率（约 30%）+ 高赔率（盈亏比 2.3-3.1）** 的趋势跟踪型策略。

## 三、策略架构

### 1. 数据要求

- **唯一数据源：5 分钟 K 线**（必须字段：datetime / open / high / low / close / volume）
- 数据库：`data_cache/kline_data.db`
- 表名约定：`{symbol}_5M_raw`（项目原有命名）

### 2. 主循环（每根 5m K 线触发一次）

```text
┌──────────────────────────────────────────────────────────┐
│ Step 1: 增量更新 czsc 分析器（O(1) 而非 O(N) 全量重建）  │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 2: 提取笔端点 + 推导中枢                              │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 3: 并行检测 4 类信号                                 │
│   - TYPE1 转向（趋势反转）                                │
│   - TYPE2 强势延续（黄金分割 1.618 倍力度）              │
│   - TYPE3 强势确认（两次回踩不破前低/高）                 │
│   - TYPE4 V 反（前 1 类失败后的反向力度突破）             │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 4: 信号聚合（多空冲突放弃；3 > 2 > 1 > 4 优先级）   │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 5: 长周期趋势过滤                                    │
│   - 5m MA240 ≈ 20 小时（等价 15m MA80）                  │
│   - 当前价 > MA × (1+0.5%) → 仅允许做多                  │
│   - 当前价 < MA × (1-0.5%) → 仅允许做空                  │
│   - 横盘区（±0.5% 内）不交易                              │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 6: 缠论"买卖点不超买"原则                            │
│   - 同方向走势内最多介入 2 次                              │
│   - 中枢更迭或信号方向反转 → 计数重置                     │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 7: 信号冷却期（6 根 5m ≈ 30 分钟）                  │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 8: 执行交易（下根 K 线开盘成交）                     │
└─────────────────────────┬────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│ Step 9: 三重止损                                          │
│   - ATR 止损（入场价 ± 2.5 × ATR14）                     │
│   - 结构止损（信号 key_level，通常是反向极点）            │
│   - 中枢破位止损（持仓方向相反的中枢边界被破）            │
└──────────────────────────────────────────────────────────┘
```

### 3. 关键参数（5m 周期调优后）

```python
strategy_params = {
    # ---- 缠论笔/信号参数 ----
    'min_bi_len': 5,                  # czsc 最小笔长度
    'golden_ratio': 1.618,            # 2 类信号黄金分割阈值
    'break_tolerance': 0.002,         # 3 类信号前低/高破位容忍度
    'v_reversal_power_ratio': 1.3,    # 4 类 V 反力度门槛

    # ---- 风控参数 ----
    'atr_period': 14,
    'atr_stop_multiplier': 2.5,       # 5m 噪声大，放宽止损
    'base_volume': 1,
    'use_structure_stop': True,
    'signal_cooldown': 6,             # 信号最小间隔 ≈ 30 分钟

    # ---- 趋势过滤（最大单一改进项）----
    'use_trend_filter': True,
    'trend_ma_period': 240,           # 5m × 240 ≈ 20 小时
    'trend_buffer': 0.005,            # ±0.5% 视为横盘

    # ---- 缠论"买卖点不超买"原则 ----
    'max_trades_per_trend': 2,

    # ---- 成交量过滤（实测拖累，默认关）----
    'use_volume_filter': False,
}
```

## 六、走过的弯路（避坑指南）

### 弯路 1：多数据源 vs 单数据源

试图通过添加 15m 数据源做趋势过滤，结果发现 SSQuant 框架**会自动把所有源对齐到最慢的时间框架**，导致 5m 数据被降采样到 15m 粒度。**信号大幅减少不是因为过滤生效，而是因为粒度错了**。

**经验：** 多数据源是个隐式的对齐黑盒。要做长周期过滤，**用 5m 数据内部合成** 更安全（`rolling(240).mean()` ≈ 20 小时）。

### 弯路 2：超时强平 + 趋势反转强平

为防"仓位被遗忘"加了两个强制平仓保险。结果反而把好仓也切掉，盈亏比从 1.98 跌到 0.72。

**经验：** 趋势策略的"亏损位置长持"和"盈利位置长持"在 K 线级别难以区分。强平保险**只在仓位规模过大、可能爆仓时**才有价值，不应该常态化触发。

### 弯路 3：成交量过滤

理论上"短均量 < 长均量 × 0.5 = 无人交易 = 噪声信号"。实测两个时间段都拖累，因为：
- 缠论 3 类信号（两次回踩不破前低/高）**本来就在低量震荡区间形成**，过滤把它们误杀
- 5/50 短长均量比阈值 0.5 太严

**经验：** "看起来合理"的过滤项必须实测，理论先验在量化里不一定对。

### 弯路 4：本金太小

初期用 10 万本金做黄金 5m（黄金单手保证金约 4.7 万），策略略一波动就强平到 0。

**经验：** 本金 vs 单手保证金至少要 5-10 倍才有足够回撤空间。10 万本金做黄金属于过度杠杆。

## 七、已知局限

1. **震荡市表现弱**（铁矿 2024 微亏 -0.75%）— 这是趋势跟踪策略的天然短板。
2. **只验证过黄金和铁矿石**，其他品种（股指、原油、农产品等）行为未知。
3. **未做实盘** — 滑点、流动性、夜盘休市、节假日等实盘细节都未验证。
4. **未做多品种组合** — 单品种亏损年份可能被其他品种抵消，下一步要验证。
5. **未做参数稳健性测试** — `trend_ma_period=240` 是经验值，更优参数可能存在。
