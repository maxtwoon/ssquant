## 一、经典缠论：3 类买卖点（缠中说禅原版）

经典缠论以"走势 → 中枢 → 买卖点"为骨架。每段走势分为三类买卖点。下面以**多头**为例（空头对称）。

### 1.1 一买（First Buy / 趋势反转底）

**定义**：下跌段（前一个走势）结束的最低点。表现为下跌中**最后一个低点**之后，价格开始反向。

**形成条件**：
- 前段是下跌走势
- 出现**底背驰**（最后一笔下跌力度 < 前一笔同向笔）或**下跌中枢的下破被反向重新攻入**
- 一买后必有一个反弹笔

**风险**：早，但容易抄到刀。胜率取决于背驰判断的准确性。

**示意图位置**：点 0

### 1.2 二买（Second Buy / 回踩不破前低）

**定义**：一买之后的第一个反弹笔（点 0→1）走完，接下来的回调笔**不破一买的低点**，那个回调的低点即为二买。

**形成条件**：
- 必须先有一买
- 反弹一段后回调
- 回调低点 > 一买的低点
- 此时反向回踩力度 < 反弹笔力度

**风险**：比一买安全得多，因为已经看到反弹动能。是缠论体系内**确定性最高**的买点之一。

**示意图位置**：点 2

### 1.3 三买（Third Buy / 突破中枢后回踩）

**定义**：一段走势形成中枢之后，价格**向上突破中枢上沿**，回调时**不破中枢上沿**，那个回调的低点即为三买。

**形成条件**：
- 已经形成一个中枢（至少 3 笔重叠区域）
- 向上一笔突破中枢的 ZG（上沿）
- 接下来的回调笔不跌破 ZG
- 实际上是新一段更高级别走势的起点

**风险**：最低。已经看到突破 + 回踩验证。但代价是价格已经离一买很远，盈亏比缩小。

**示意图位置**：点 6

## 二、本项目的 4 类信号（`detect_typeN_signal`）

项目里的实现是缠中说禅 3 类买点的**变体 + 拓展**，多了一个 V 反信号。

| 项目实现 | 经典对应 | 关键判据 |
|---|---|---|
| **TYPE1**（趋势反转）| ≈ 一买 + 二买的合并 | 最近 4 个端点呈 L-H-L-H（多）或 H-L-H-L（空），且 **低点抬高 + 高点突破** |
| **TYPE2**（强势延续）| ≈ 突破中段加仓 | 同方向相邻两笔，**后笔力度 ≥ 前笔 × 1.618**（黄金分割）|
| **TYPE3**（强势确认）| ≈ 三买 | 连续 **3 个低点不破前低**（容忍 0.2%），最后一笔向上 |
| **TYPE4**（V 反）| 无经典对应 | 前面出现过 1 类反向信号后，反向力度突破 |

### 关键差异

1. **项目的 TYPE1 比经典一买"晚"**：经典一买是下跌段最低点（点 0），项目的 TYPE1 要等"低点抬高 + 高点突破"两个确认，相当于已经走到点 3 才确认 — 更稳但入场价更高。

2. **项目的 TYPE2 ≠ 经典二买**：经典二买是结构性买点（一买后第一回踩），项目的 TYPE2 是**力度型加仓**（同方向连续两笔，后笔猛）。两者用途不同：经典二买帮你建仓，项目 TYPE2 帮你加仓。

3. **项目的 TYPE3 ≈ 经典三买**，但要求 **3 个低点都不破前低**（而非经典 2 次回踩）。更严格，胜率高但触发少。

4. **项目独有的 TYPE4 (V 反)**：经典缠论没有，是为了捕捉"假突破后的反向冲击"。胜率较低（强度 0.5），仓位也减半（volume_ratio=0.5）。

### 信号聚合优先级（`aggregate_signals`）

`TYPE3 > TYPE2 > TYPE1 > TYPE4`，多空冲突放弃。优先级反映**确定性而非时间**：3 类信号的结构最确认。

## 四、实践建议

| 场景 | 推荐用 | 原因 |
|---|---|---|
| 抄底（高风险高收益） | 经典一买 | 离底最近，但需要背驰判断 |
| 趋势确认建仓 | 经典二买 / 项目 TYPE1 | 看到反弹动能后入场 |
| 突破后追单 | 经典三买 / 项目 TYPE3 | 中枢突破已确认，进场点最稳 |
| 同方向加仓 | 项目 TYPE2 | 利用力度比叠加趋势 |
| 假突破修正 | 项目 TYPE4 | 用得少，仓位减半 |

在本项目的 2024 黄金 5m 实盘信号统计里（来自 2026-05 的回测）：
- TYPE3_LONG 信号最多（694 个，胜率 54.6%）
- TYPE1_LONG 95 个，胜率 51.6%
- TYPE2 / TYPE4 全年 0 次触发（阈值定得太高）

→ 实战中**TYPE3（三买）是最稳定的赚钱机制**，跟经典缠论的理论结论一致。
